Commands

All available CSMS commands. Type them anywhere in your adventure input.

How to use commands: Type any command directly into the adventure input field, the same way you'd type an action or dialogue. CSMS detects the command automatically — you don't need to be on any special screen.

Character Management

CommandDescription
/csms create [name]Creates a new character sheet. The first character created is the player. All subsequent characters are NPCs.
/csms statsRefreshes your character sheet display.
/csms stats [name]Refreshes a specific character's sheet.
/csms syncForces an immediate sync of your story card edits to state.
/csms sync [name]Forces immediate sync for a specific character's card.

Development Points

CommandDescription
/csms dp/checkShows your available DP and current stats.
/csms dp/STR:2,INT:1Spends DP across stats in one command. Any combination of stats works. Total DP spent must not exceed available DP.

Resetting

CommandDescription
/csms reset/character/[name]Removes a specific character's sheet entirely — both the story card and state data.
/csms reset/pendingClears all active pending states. Use this if a multi-step process (roll, ordinance) gets stuck.
/csms cleanupRemoves orphaned 📋 story cards that no longer have matching character data.

Rolls

CommandDescription
/csms roll [action]Triggers a dice roll for the described action. AI determines the relevant stat. Script rolls. Result feeds back into the narrative.
/csms roll vs [name]Opposed roll — both sides roll, higher wins, damage applied automatically.

Ordinance

CommandDescription
/csms ordinance/Caller/Name/TargetExecutes an Ordinance. Target is optional for non-damaging Ordinances.

Notes

Commands work anywhere in your input

You can type a command as part of regular text and it will still be detected. For example:

> You check your status. /csms stats

Both the action and the command will work. The AI sees your action; CSMS intercepts the command.

Names are case-insensitive

/csms reset nikolai and /csms reset Nikolai and /csms reset NIKOLAI all do the same thing.

Punctuation is stripped from names

If your input contains punctuation near the name (like quotation marks), CSMS strips it automatically. /csms create "Diana" creates a character named Diana, not "Diana".

Multiplayer behavior

In multiplayer with Third Person mode:

  • /csms stats with no name shows your character, not necessarily the first character created
  • /csms sync with no name syncs your character
  • Any player can create or reset any character — there are no ownership restrictions
