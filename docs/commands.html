Commands

All available CSMS commands. Type them anywhere in your adventure input.

How to use commands: Type any command directly into the adventure input field, the same way you'd type an action or dialogue. CSMS detects the command automatically â€” you don't need to be on any special screen.

Character Management

Command Description
/csms create [name] Creates a new character sheet. The first character created is marked as the player character. All subsequent characters are NPCs. Replace [name] with the character's name.
/csms stats Refreshes and displays your character sheet (player character). Use this if your sheet seems out of date or not showing correctly.
/csms stats [name] Refreshes a specific character's sheet by name. Useful for checking NPC sheets.

Syncing

Command Description
/csms sync Forces an immediate sync of your story card edits to state. Normally this happens automatically on the next action â€” use this if you want it to update right now.
/csms sync [name] Forces immediate sync for a specific character's card.

Resetting

Command Description
/csms reset [name] Removes a specific character's sheet entirely â€” both the story card and the state data. The character's name is case-insensitive.
/csms reset Removes ALL character sheets. Use with caution â€” this cannot be undone.
/csms cleanup Scans for orphaned ðŸ“‹ story cards that no longer have matching character data and removes them. Run this if you see stale cards in your story card list.

Notes

Commands work anywhere in your input

You can type a command as part of regular text and it will still be detected. For example:

> You check your status. /csms stats

Both the action and the command will work. The AI sees your action; CSMS intercepts the command.

Names are case-insensitive

/csms reset nikolai and /csms reset Nikolai and /csms reset NIKOLAI all do the same thing.

Punctuation is stripped from names

If your input contains punctuation near the name (like quotation marks), CSMS strips it automatically. /csms create "Diana" creates a character named Diana, not "Diana".

Multiplayer behavior

In multiplayer with Third Person mode:

  • /csms stats with no name shows your character, not necessarily the first character created
  • /csms sync with no name syncs your character
  • Any player can create or reset any character â€” there are no ownership restrictions
