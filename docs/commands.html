Reference

Commands

All available CSMS commands. Type them anywhere in your adventure input.

How to use commands: Type any command directly into the adventure input field, the same way you'd type an action or dialogue. CSMS detects and intercepts the command — you don't need to be on any special screen.

Format: Arguments are separated by / — not spaces. For example: /csms create/John Smith, not /csms create John Smith.

Character Management

CommandDescription
/csms create/NameCreates a new character sheet. The first character created becomes the player. All subsequent characters are NPCs. Full names with spaces work — /csms create/John Smith is valid.
/csms checkRefreshes your own character sheet display.
/csms check/NameRefreshes a specific character's sheet by name.
/csms update/statsForce-syncs your story card edits to state right now instead of waiting for the next action.
/csms update/stats/NameForce-syncs a specific character's card.
/csms update/notes/NameRe-parses a character's inventory and ordinance blocks from their sheet notes.

Development Points

CommandDescription
/csms dp/checkShows your available DP and current stats.
/csms dp/STR:2,INT:1Spends DP across stats in one command. Any combination of stats works. Total spent must not exceed available DP.

Resetting

CommandDescription
/csms reset/character/NameRemoves a specific character's sheet entirely — both the story card and all state data. Cannot be undone.
/csms reset/pendingClears all active pending states. Use this if a roll or ordinance sequence gets stuck mid-process.
/csms cleanupRemoves orphaned 📋 story cards that no longer have matching character data in state.

Rolls

CommandDescription
/csms roll/Caller/actionTriggers a dice roll. The AI determines the most relevant stat for the described action. Result feeds back into the narrative.
/csms roll/Caller/action/OpponentOpposed roll — both sides roll, higher wins. If Opponent is a character with a sheet, their stats factor in. XP awarded to the winner.

Ordinance

CommandDescription
/csms ordinance/Caller/OrdinanceName/TargetExecutes an Ordinance. Target is optional for non-damaging Ordinances. All three names must match existing character sheets (except Target on non-damaging).

Notes

Commands work anywhere in your input

You can mix a command with regular text — both fire. The AI sees your prose; CSMS intercepts the command:

I pause to take stock. /csms check

Names are case-insensitive

/csms check/nikolai and /csms check/Nikolai both work. Just be consistent with spacing — John Smith and john smith match, but JohnSmith does not.

Punctuation is stripped

Trailing punctuation like quotes and periods are stripped automatically from names. /csms create/"Diana" creates a character named Diana.

Re-running a command

Commands fire during the input step — before the AI generates anything. If you delete only the AI's response and press Continue, the command won't fire again. To re-run it, delete both the AI response and your original input, then retype and submit.

Multiplayer

/csms check and /csms update/stats with no name always act on the character of the player who typed the command, not the first character created.
