Reference

Commands

All available CSMS commands. Type them anywhere in your adventure input.

How to use commands: Type any command directly into the adventure input field, the same way you'd type an action or dialogue. CSMS detects the command automatically — you don't need to be on any special screen. Commands can even be combined with normal text in the same input.

Character Management

Command Description
/csms create/[name] Creates a new character sheet. The first character created is marked as the player character. All subsequent characters are NPCs.
/csms stats Refreshes and displays your character sheet. Use this if your sheet seems out of date or not showing correctly.
/csms stats/[name] Refreshes a specific character's sheet by name. Useful for checking NPC sheets.
/csms reset/[name] Removes a specific character's sheet entirely — both the story card and the state data. Case-insensitive.
/csms reset Removes ALL character sheets. Use with caution — this cannot be undone.
/csms cleanup Scans for orphaned 📋 story cards that no longer have matching character data and removes them. Run this if you see stale cards in your story card list.

Update

Command Description
/csms update/stats/[name] Reads the Entry field of a character's story card and syncs their stats to state. Run this after manually editing stats on the card.
/csms update/inventory/[name] Reads the Notes field of a character's story card and syncs their inventory to state. Run this after editing the [Inventory] block in the card.

Dice & Rolls

Command Description
/csms roll/[action] Initiates a roll for an action. CSMS asks the AI which stat is most relevant, then rolls 1d20 + stat modifier. The result feeds back into the next AI response. Replace [action] with what you're attempting — e.g. /csms roll/pick the lock.

Ordinance

Command Description
/csms ordinance/[name]/[target] Triggers an Ordinance against a target. CSMS finds the Ordinance story card by name, feeds its entry to the AI, and executes the outcome mechanically — including rolls and damage if the Ordinance warrants it. Replace [name] with the Ordinance card title and [target] with the target character's name.
Ordinance example: If you have an Ordinance card titled Quick Strike and want to use it against a Guard — type /csms ordinance/Quick Strike/Guard. CSMS handles the rest.

Notes

Commands work anywhere in your input

You can type a command alongside regular text and both will work. For example:

> You size up the guard and make your move. /csms ordinance/Quick Strike/Guard

The AI sees your action. CSMS intercepts the command.

Names are case-insensitive

/csms reset/nikolai and /csms reset/Nikolai both do the same thing.

Punctuation is stripped from names

CSMS strips punctuation from name parameters automatically. /csms create/"Diana" creates a character named Diana.

Multiplayer behavior

In multiplayer with Third Person mode:

  • /csms stats with no name shows your character
  • /csms roll always rolls for the active player
  • Any player can create or reset any character — there are no ownership restrictions
