Reference

Modules

Every system in CSMS, in one place. Each module can be toggled independently via the ⚙️ CSMS CFG card.

Character Sheets

The foundation of everything. Every character — player or NPC — gets a 📋 story card tracking their stats, HP, level, XP, and proficiency. The AI sees a compact summary of every active character on every action.

Stats use D&D 5e as a base: STR, DEX, CON, INT, WIS, CHA. Each has a modifier calculated from the score. The card is editable directly from the Story Cards panel — the script validates and syncs on the next action.

Name: Nikolai | Level: 3 | XP: 450
HP: 18/20 | AC: 15 | Speed: 30ft
Proficiency: +2

STR: 15 (+2)  DEX: 12 (+1)  CON: 14 (+2)
INT: 10 (+0)  WIS: 11 (+0)  CHA: 13 (+1)

Toggle: MODULE_CHARACTER_SHEETS — includes the tag system and AI context injection. This is the core module. Disabling it disables everything else.

Tag System

Add [CSMS] to the start of any story card title. When that character's name appears in recent story history, CSMS asks the AI to generate appropriate stats from the card description and creates a full sheet automatically. The tag is removed from the original card — your lore card stays intact.

Name:     [CSMS] Mira
Type:     Character
Entry:    Mira is a battle-hardened mercenary with fifteen years
          of combat experience. Fast and agile, relies on speed
          over strength. Completely unshakeable under pressure.
Triggers: Mira, mercenary
Two-step render. On the first action after detection, a placeholder card appears. On the second action, real stats fill in. This is an AID rendering limitation — intentional behavior, not a bug.

Toggle: Part of MODULE_CHARACTER_SHEETS.

Roll System

Dice rolls with stat modifiers. The AI determines the most relevant stat for the action — the script rolls, resolves the outcome, and injects the result back into the narrative.

/csms roll/Caller/action
/csms roll/Caller/action/Opponent

Flow: player submits roll → context hook asks AI for the relevant stat → output hook catches the stat, resolves the roll, injects a narrative instruction → player sees the roll breakdown → Continue → AI narrates the outcome.

For opposed rolls, both sides roll. Higher wins. XP is awarded to the winner. If opponent is a character name with a sheet, their stats factor into the roll.

Toggle: MODULE_COMBAT

Ordinance System

Named special abilities written as story cards. The world maker writes the rules in plain language — a combat technique, a signature move, a hacking method, anything. The script executes, rolls dice, detects outcomes in the AI's narrative, and applies real HP consequences.

/csms ordinance/Caller/OrdinanceName/Target

Flow: classify (DAMAGING or NONDAMAGING) → roll if needed → AI narrates freely using the ordinance card entry as its instructions → script scans narrative for damage-tier words → HP applied to caller, target, or both depending on what the AI wrote.

Characters can be restricted to specific Ordinances via their [Ordinances] block. An empty block means access to all ordinances in the scenario.

Toggle: MODULE_ORDINANCE — see the full Ordinance guide.

Inventory System

Item tracking per character, stored in the [Inventory] block of the character sheet. Pick up, drop, give, throw — using natural act_ prefixed input. The prefix is stripped before the AI sees anything.

I walk over and act_grab the briefcase.
act_give the documents to Sarah.
act_drop the wallet on the floor.
I turn and act_throw the bottle at the guard.

Items are editable directly in the story card. Synced automatically on the next action.

Toggle: MODULE_INVENTORY — see the full Inventory guide.

Leveling System

Automatic XP tracking and level progression. XP accumulates passively and through specific actions. On level up: max HP increases by HP_PER_LEVEL, Development Points are awarded, and proficiency may increase depending on configured thresholds.

XP SourceConfig KeyDefault
Per action takenXP_PER_ACTION1
Per ordinance executedXP_PER_ORDINANCE5
Per successful rollXP_PER_ROLL_SUCCESS3
Per failed rollXP_PER_ROLL_FAIL1
Per damage dealtXP_PER_HIT3
Per damage receivedXP_PER_DAMAGE_RECEIVED2

Set any source to 0 to disable it. NPCs level up too — DP is auto-allocated on their behalf.

/csms dp/check              — see available DP and current stats
/csms dp/STR:2,INT:1        — spend DP across stats in one command

Action System

Natural language mechanics triggered by act_ prefixed keywords. Part of the Inventory and Leveling modules — no separate toggle. The prefix and keyword are stripped before the AI sees the input, so the story reads naturally.

Debug UI

A persistent debug log card — 🔧 CSMS DEBUG — visible in your Story Cards panel. Only active when DEBUG_MODE: true in the CFG card. The log buffers in state and flushes on every output hook.

To run a debug command, type it into the Entry field of the debug card and take an action:

CommandEffect
CLEAR LOGWipes the debug log
CLEAR:ordinanceClears the ordinance pending state
CLEAR:rollClears the roll pending state
SET:csmsPending.ordinance.step=NManually sets the ordinance step to N
SET:csmsPending.roll.step=NManually sets the roll step to N
Always disable before publishing. Set DEBUG_MODE: false in your ⚙️ CSMS CFG card. Debug mode logs every action — useful while building, noise for players.
