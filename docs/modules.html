Reference

Modules

Every system in CSMS, in one place. Each module can be toggled independently.

Character Sheets

The foundation of everything. Every character — player or NPC — gets a 📋 story card tracking their stats, HP, level, XP, and proficiency. The AI sees a compact version of every active character's stats on every action.

Stats use D&D 5e as a base: STR, DEX, CON, INT, WIS, CHA. Each has a modifier calculated from the score. The card is editable directly from the Story Cards panel — the script validates and syncs automatically on the next action.

Name: Nikolai | Level: 3 | XP: 450
HP: 18/20 | AC: 15 | Speed: 30ft
Proficiency: +2

STR: 15 (+2)  DEX: 12 (+1)  CON: 14 (+2)
INT: 10 (+0)  WIS: 11 (+0)  CHA: 13 (+1)

Toggle: MODULE_LEVELING controls XP/level display. Character sheets themselves are always on.

Tag System

Add [CSMS] to the start of any story card title. When that character's name appears in recent story history, CSMS asks the AI to generate appropriate stats from the card's description — and creates a full character sheet automatically.

The tag is removed from the original card after generation. Your lore card stays intact. A separate 📋 sheet is created.

Title:  [CSMS] Mira
Entry:  Mira is a battle-hardened mercenary with fifteen years
        of combat experience. Fast and agile, relies on speed
        over strength. Completely unshakeable under pressure.
Two-step render. On the first action after detection, a placeholder card appears. On the second action, real stats fill in. This is an AID rendering limitation — intentional behavior, not a bug.

Roll System

Dice rolls with stat modifiers. Supports standard roll notation, advantage, and disadvantage.

/csms roll [action]        — AI determines stat, script rolls
/csms roll vs Name         — opposed roll, higher wins, damage applied

Flow: player inputs action → context hook asks AI for stat → output hook catches stat, resolves roll, injects narrative instruction → player sees roll breakdown in notification → Continue → AI narrates outcome.

XP is awarded to the winner of opposed rolls.

Toggle: MODULES.COMBAT

---

Ordinance System

Named special abilities written as story cards. The world maker writes the rules in plain language. The script executes, rolls dice, detects outcomes in AI narrative, and applies damage.

/csms ordinance/Caller/OrdinanceName/Target

Flow: classify (DAMAGING/NONDAMAGING) → roll if needed → AI narrates outcome → script reads narrative for damage words and applies HP. The ordinance card's entry is injected directly at narration step — AI reads the actual rules, not just the name.

Characters can be restricted to specific Ordinances via their [Ordinances] block. Empty block = access to all.

Toggle: MODULES.ORDINANCE — see the full Ordinance guide.

Inventory System

Item tracking per character, stored in the character sheet Notes field. Pick up, drop, give, throw — using natural act_ prefixed input.

I walk over and act_grab the briefcase.
act_give the documents to Sarah.
I turn and act_throw the bottle at the guard.

Items live in the [Inventory] block of the character sheet. Editable directly. Synced automatically.

Toggle: MODULES.INVENTORY — see the full Inventory guide.

Leveling System

Automatic XP tracking and level progression. XP is awarded passively every action, and additionally for rolls, combat, Ordinances, and learning. On level up: max HP increases, Development Points (DP) are awarded, proficiency may increase.

XP SourceConfig Key
Per action takenXP_PER_ACTION
Per ordinance executedXP_PER_ORDINANCE
Per successful rollXP_PER_ROLL_SUCCESS
Per failed rollXP_PER_ROLL_FAIL
Per damage dealtXP_PER_HIT
Per damage receivedXP_PER_DAMAGE_RECEIVED
Per killing blowXP_PER_KILL
Per learn actionXP_PER_LEARN

Set any source to 0 to disable it. NPCs level up too — DP is auto-allocated on their behalf.

/csms dp/check              — see available DP and current stats
/csms dp/STR:2,INT:1        — spend DP across stats in one command

Toggle: MODULES.LEVELING

Action System

Natural language mechanics triggered by act_ prefixed keywords in player input. The prefix is stripped before the AI sees anything — the AI narrates freely, the script tracks the consequence.

Debug UI

A persistent debug log card — 🔧 CSMS DEBUG — visible in your Story Cards panel. Enable via DEBUG_MODE: true in your config card. The log buffers in state and flushes every output hook.

Debug card supports commands in its Notes field:

CommandEffect
CLEAR_LOGWipes the debug log
CLEAR:ordinanceClears the ordinance pending state
CLEAR:rollClears the roll pending state
SET:path=valueSets a whitelisted state path directly — e.g. SET:csmsPending.ordinance.step=3
Always disable before publishing. DEBUG_MODE: false in your ⚙️ CSMS CFG card. Debug mode logs every action — useful for you, noise for players.
