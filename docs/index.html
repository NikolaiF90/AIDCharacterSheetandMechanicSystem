Ordinance System

Rules that actually
do something.

An Ordinance is a story card with a rule written in plain language. When triggered, CSMS feeds it to the AI — and the outcome isn't just narrated. It's executed. Stats update. Damage applies. The story reflects what actually happened.

Philosophy

Three jobs.
Three owners.

Most mechanics systems either fight the AI — hardcoding outcomes the story should decide — or do too little, leaving everything to narrative luck. CSMS draws a clear line between three responsibilities and makes sure each one stays in the right hands.

Player

Decides when mechanics apply. Invokes rolls. Triggers Ordinances. Agency stays with the person playing the story.

+
AI

Decides what rules apply. Reads Ordinance entries. Judges context. Narrates outcomes with meaning.

+
Script

Handles the numbers. Rolls dice. Applies damage. Updates HP. Math is never left to chance or imagination.

Features

Everything you need.
Nothing you don't.

⚔️

Ordinance System

Write rules in plain language on a story card. Script feeds it to AI, AI interprets it, script executes the numbers. Damage applies. Stats update. The story reflects reality.

🎲

Dice Engine

Full roll notation — 1d20+STR, advantage, disadvantage. Stat modifiers calculated automatically. Player triggers rolls explicitly. No guessing, no luck-based narration.

📋

Character Sheets

Dynamic story cards for every character — player and NPCs. Always visible, always in sync. Edit values directly in the card. Script validates and rewrites cleanly.

🏷️

Tag System

Add [CSMS] to any story card. When that character appears in the story, a sheet is auto-generated from their description. No manual stat assignment needed.

👥

Multiplayer Ready

Sheets auto-create when players join. Active player detection. Name change sync. Works out of the box with the Third Person plot component.

🎒

Inventory

Item tracking per character, stored inside their sheet card. Human readable, AI readable, script parseable. Player edits freely — script syncs on command.

🔔

NotifyThem

Built-in notification system. Script communicates to player in clean brackets without breaking immersion. Also available as a standalone system for any AID script.

🧩

Modular

Every system is a toggle. Use only what you need — sheets without combat, combat without Ordinance, or the full stack. Your scenario, your call.

Quick Start

Up and running
in 5 minutes.

01

Copy the Library

Grab library.js from the GitHub repo and paste the entire contents into your scenario's Library tab.

02

Add the Hooks

Three lines, three tabs. Input, Context, Output. Each gets one call to CSMS(hook).

CSMS("input");   // Input tab
CSMS("context"); // Context tab
CSMS("output");  // Output tab
03

Create a Character

In your adventure, type /csms create YourName. Your character sheet appears instantly.

04

Write Your First Ordinance

Create a story card. Give it a name. Write the rule in plain language. Trigger it with /csms ordinance/Name/Character. That's it.
